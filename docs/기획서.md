## 1. 한 줄 정의

**"이 가격이면 사도 되나?"에 즉시 답해주는 생활물가 판정 서비스.**

계란·호박·배추 같은 장바구니 품목마다 평균 가격을 기준선으로 잡고, 오늘 가격이 그보다 싼지 비싼지를 색과 아이콘으로 한눈에 보여준다. 웹과 모바일 앱을 동시에 제공한다.

---

## 2. 문제 정의

| 현재 상황 | 사용자의 어려움 |
|---|---|
| 뉴스는 "물가 ○% 상승" 같은 지수만 보도 | 내가 사는 품목이 지금 비싼지 알 수 없음 |
| KAMIS 등 공공 데이터는 존재하지만 표 위주 | 일반 소비자가 찾아보기 어렵고 해석이 안 됨 |
| 마트 가격표만 봐서는 기준이 없음 | "계란 한 판 8,900원"이 싼 건지 바가지인지 판단 불가 |
| 가격은 계절·작황 따라 크게 출렁임 | 언제 사는 게 유리한지 감으로만 판단 |

핵심 통찰: 사용자에게 필요한 것은 **가격 데이터가 아니라 "판정"**이다. 숫자를 보여주는 서비스는 많지만, "지금 싸다 / 평소 수준이다 / 비싸니 기다려라"를 말해주는 서비스는 없다.

---

## 3. 타깃 사용자

- **주 타깃**: 장보기를 직접 하는 20~50대 소비자. 가격에 민감하고, 마트/시장/온라인을 오가며 비교 구매함.
- **부 타깃**:
  - 소규모 식당·카페 운영자 — 식자재 매입 타이밍 판단
  - 가계부·절약 커뮤니티 사용자 — 물가 정보 공유 목적

### 대표 페르소나

> **김서연 (34, 맞벌이, 주 1회 장보기)**
> 마트에서 애호박이 2,500원인 걸 보고 "원래 이 가격이었나?" 싶어 검색하지만 답을 못 찾는다. 앱을 열어 애호박이 "평균보다 38% 비쌈 🔺"인 걸 확인하고 이번 주는 다른 채소로 대체한다.

---

## 4. 핵심 가치 제안

1. **판정 우선**: 모든 품목에 `저렴 / 보통 / 비쌈` 3단계 배지를 부여. 숫자를 몰라도 색만 보면 됨.
2. **한눈에 보는 등락**: 어제·지난주·지난달 대비 상승 🔺 / 하락 🔻 / 보합 ➖ 을 목록에서 즉시 확인.
3. **기준선 제시**: 품목별 평균 가격(평시 가격)을 항상 함께 표기 → 매장에서 가격표와 바로 비교 가능.
4. **장보기 최적화**: "지금 사면 좋은 품목" / "기다리면 좋은 품목"을 추천.

## 5. 주요 기능

### 5.1 MVP (1차 출시 범위)

#### ① 오늘의 물가 대시보드 (홈)
- 품목 카드 그리드: 품목명 · 오늘 평균 소매가 · 평시 가격 · 등락 아이콘 · 판정 배지
- 정렬: 많이 오른 순 / 많이 내린 순 / 가나다순 / 즐겨찾기 우선
- 카테고리 필터: 채소 · 과일 · 축산 · 수산 · 곡물 · 가공식품
- 상단 요약 밴드: "오늘 32개 품목 중 🔻 하락 12 · 🔺 상승 9 · ➖ 보합 11"

#### ② 품목 상세
- 가격 추이 차트 (1주 / 1개월 / 1년 / 3년)
- 평시 가격 밴드(정상 범위)를 차트에 음영으로 표시 → 현재가가 밴드 위인지 아래인지 시각화
- 단위 정보 명확화 (예: 계란 특란 30구, 배추 1포기 2.5~3kg)
- 소매가/도매가 구분, 지역별 가격 (시·도 단위)
- "역대 이 시기 평균" 비교 (계절성 반영: 7월의 배추는 원래 비싸다)

#### ③ 판정 시스템 (서비스의 심장)
- 3단계 배지: 🟢 저렴 · ⚪ 보통 · 🔴 비쌈
- 판정 문구 예시: "계란(특란 30구) 오늘 6,480원 — 평시 6,900원보다 **6% 저렴** 🟢 사기 좋은 시기"

#### ④ 즐겨찾기 (나의 장바구니)
- 자주 사는 품목 등록 → 홈 상단 고정
- 내 장바구니 총 물가 지수: "내 장바구니, 지난달보다 4.2% 올랐어요"

#### ⑤ 검색
- 품목명 자동완성 (별칭 지원: "애호박=호박", "달걀=계란")

### 5.2 2차 확장 (출시 후)

| 기능 | 설명 |
|---|---|
| 가격 알림 | 즐겨찾기 품목이 "저렴" 구간 진입 시 푸시 알림 |
| 주간 리포트 | "이번 주 장보기 브리핑" 푸시/이메일 |
| 대체 품목 추천 | 애호박이 비싸면 "주키니는 평균 수준이에요" |
| 김장 지수 등 시즌 기획 | 김장철 재료 묶음 물가, 명절 차례상 물가 |
| 사용자 제보 가격 | 실제 구매 가격 제보 → 지역 체감가 보정 (신뢰도 검증 필요) |
| 위젯 | iOS/Android 홈 화면 위젯: 즐겨찾기 품목 등락 요약 |

## 6. 데이터 소스 및 수집 전략

| 출처 | 제공 데이터 | 갱신 주기 | 비고 |
|---|---|---|---|
| **KAMIS** (aT 농수산물유통정보) OpenAPI | 농축수산물 도·소매 일일 가격, 지역별 | 매일 (영업일) | 핵심 소스. 품목·등급·단위 체계 보유 |
| **한국소비자원 참가격** | 가공식품·생필품 판매 가격 | 주간 | 라면·우유·휴지 등 공산품 확장 시 |
| **축산물품질평가원** | 축산물 경락/소비자 가격 | 매일 | 소·돼지·닭·계란 보강 |
| **KOSIS (통계청)** | 소비자물가지수(CPI) 품목별 | 월간 | 장기 추세·검증용 |

### 수집 파이프라인
- 일 1~2회 배치 수집 (KAMIS 발표 시각 이후, 평일 15시경 + 보정 1회)
- 수집 실패 대비: 재시도 → 전일 데이터 유지 + "N일 전 기준" 표시 (오래된 데이터를 최신처럼 보여주지 않음)
- 원본 데이터는 그대로 적재(raw), 정제 테이블 분리 → 판정 로직 변경 시 재계산 가능

## 7. 판정 로직 (초안)

핵심 원칙: **계절성을 반영한 "평시 가격"**과 비교한다. 단순 연평균과 비교하면 여름 배추는 항상 "비쌈"으로 표시되는 오류가 생긴다.

```
평시가격(품목, 오늘) = 최근 3년간 같은 시기(±2주) 가격의 중앙값
                       (물가 상승분 보정: CPI 또는 최근 1년 추세로 스케일링)

편차율 = (오늘 가격 − 평시가격) / 평시가격

판정:
  편차율 ≤ −10%          → 🟢 저렴
  −10% < 편차율 < +10%   → ⚪ 보통
  편차율 ≥ +10%          → 🔴 비쌈
```

- 등락 표시는 별도 축: 전일/전주 대비 ±2% 이상 변동 시 🔺/🔻, 미만은 ➖
- 임계값(±10%, ±2%)은 품목별 변동성에 따라 조정 여지 있음 (예: 변동성 큰 채소는 ±15%)
- 판정 근거는 상세 화면에서 투명하게 공개 ("최근 3년 7월 중순 중앙값 기준")

```
홈 (오늘의 물가 대시보드)
├─ 품목 카드 → 품목 상세
│               ├─ 가격 추이 차트
│               ├─ 지역별 가격
│               └─ 즐겨찾기 추가
├─ 카테고리 탭 (채소/과일/축산/수산/…)

└─ 마이
    ├─ 즐겨찾기(나의 장바구니)
    ├─ 알림 설정 (2차)
    └─ 설정 (지역, 단위 표기)
```

### 홈 카드 레이아웃 (개념)

```
┌─────────────────────────────┐
│ 🥚 계란 (특란 30구)          │
│ 6,480원   🔻 −1.2% (전일)   │
│ 평시 6,900원                 │
│ 🟢 저렴 — 사기 좋은 시기      │
└─────────────────────────────┘
```

## 9. 기술 스택 제안

웹/앱 동시 출시가 요구사항이므로 **단일 코드베이스 크로스플랫폼**을 권장.

| 레이어 | 선택 | 이유 |
|---|---|---|
| 클라이언트 | **Flutter** (iOS·Android·웹) | 한 코드로 3개 플랫폼, 차트·커스텀 UI 강점. 대안: Expo(React Native)+React 웹 |
| 백엔드 API | Kotlin(Ktor) 또는 Node(NestJS) | 수집 배치와 API 서버 통합 운영 |
| DB | PostgreSQL | 시계열 가격 데이터 + 품목 마스터. 규모 커지면 TimescaleDB 확장 |
| 배치/스케줄러 | 서버 내 크론 or GitHub Actions | 일 1~2회 수집이므로 가볍게 시작 |
| 인프라 | 초기: 단일 VPS/Cloud Run + 관리형 DB | MVP는 트래픽 예측 불가 → 저비용 시작 |
| 캐싱 | 일 단위 데이터 특성상 CDN/HTTP 캐시로 충분 | Redis는 필요해질 때 도입 |

> 참고: 데이터가 하루 1~2회만 바뀌는 서비스이므로 실시간 인프라가 필요 없음. "가볍게 만들고 캐시를 세게 건다"가 비용 전략의 핵심.

## 10. MVP 품목 선정 (초기 30~40개)

소비 빈도가 높고 KAMIS 데이터 품질이 좋은 품목부터.

- **채소**: 배추, 무, 양파, 대파, 마늘, 애호박, 오이, 상추, 시금치, 감자, 고구마, 당근, 깻잎, 청양고추
- **과일**: 사과, 배, 바나나, 딸기(시즌), 수박(시즌), 포도, 감귤(시즌), 토마토
- **축산**: 계란(특란 30구), 돼지 삼겹살, 돼지 목살, 소 등심(한우/수입 구분), 닭고기(생닭 1kg)
- **수산**: 고등어, 오징어, 갈치, 김
- **곡물/기타**: 쌀(20kg), 두부, 콩나물

## 11. 로드맵

| 단계 | 기간(안) | 내용 |
|---|---|---|
| **M1. 데이터 기반** | 2~3주 | KAMIS 연동, 수집 파이프라인, 품목 마스터, 판정 로직 v1, 3년치 백필 |
| **M2. MVP 클라이언트** | 3~4주 | 홈 대시보드, 품목 상세(차트), 검색, 즐겨찾기 — 웹 우선 배포 |
| **M3. 앱 출시** | 2~3주 | 스토어 심사 대응, 앱 최적화, 위젯 기초 |
| **M4. 리텐션 기능** | 이후 | 가격 알림, 주간 리포트, 대체 품목 추천 |

M2에서 **웹을 먼저 배포**하는 이유: 스토어 심사 없이 즉시 사용자 피드백을 받을 수 있고, Flutter 웹 빌드로 추가 비용이 거의 없음.

## 12. 성공 지표 (KPI)

- **핵심**: 주간 재방문율 (장보기 주기와 일치하는 주 1회 이상 방문)
- 즐겨찾기 등록 사용자 비율 (개인화 = 리텐션의 선행 지표)
- 홈 → 품목 상세 진입률 (판정만 보고 끝나는지, 더 궁금해하는지)
- (2차) 알림 수신 → 앱 진입 전환율

## 13. 리스크와 대응
